# FloraMarket — Маркетплейс ботанических растений

Красивый маркетплейс для покупки и продажи ботанических растений с VIP/срочными объявлениями, админ-панелью и интеграцией ЮKassa.

## Возможности

- **Каталог растений** с фильтрами, поиском и анимациями
- **Размещение объявлений** — бесплатное, срочное и VIP
- **Платные тарифы** — срочная продажа и VIP-размещение
- **Модерация** объявлений перед публикацией
- **Админ-панель** — пользователи, модерация, платежи, настройка цен
- **Оплата через ЮKassa** — шаблон готов, нужно только добавить токены
- **Юридические страницы** — оферта, политика, контакты (требования ЮKassa)

## Требования

- [Node.js](https://nodejs.org/) 18 или новее
- npm

## Установка

```bash
# 1. Установите Node.js с https://nodejs.org/ (LTS версия)

# 2. Установите зависимости
npm install

# 3. Создайте файл окружения
copy .env.example .env

# 4. Инициализируйте базу данных
npx prisma db push
npm run db:seed

# 5. Запустите сайт
npm run dev
```

Сайт откроется на http://localhost:3000

## Подключение ЮKassa (после модерации)

1. Зарегистрируйтесь на [yookassa.ru](https://yookassa.ru)
2. Пройдите модерацию сайта (все юридические страницы уже готовы)
3. Получите `shopId` и `secretKey` в личном кабинете
4. Добавьте в `.env`:

```env
YOOKASSA_SHOP_ID="ваш_shop_id"
YOOKASSA_SECRET_KEY="ваш_secret_key"
YOOKASSA_MODE="live"
NEXT_PUBLIC_SITE_URL="https://ваш-домен.ru"
```

5. Настройте webhook URL в ЮKassa:
   ```
   https://ваш-домен.ru/api/payments/webhook
   ```

6. Заполните реквизиты компании в `.env`:
   ```env
   COMPANY_NAME="ИП Иванов И.И."
   COMPANY_INN="ваш_инн"
   COMPANY_OGRN="ваш_огрн"
   COMPANY_ADDRESS="ваш адрес"
   COMPANY_EMAIL="info@ваш-домен.ru"
   COMPANY_PHONE="+7 (999) 123-45-67"
   ```

## Админ-панель

Доступна по адресу `/admin` для пользователей с ролью ADMIN.

- **Обзор** — статистика сайта
- **Модерация** — одобрение/отклонение объявлений
- **Пользователи** — база всех пользователей
- **Платежи** — история оплат
- **Настройки** — цены за срочное и VIP размещение

## Деплой

Для продакшена рекомендуется:
- [Vercel](https://vercel.com) — для Next.js
- PostgreSQL вместо SQLite (измените `provider` в `prisma/schema.prisma`)
- HTTPS обязателен для ЮKassa
